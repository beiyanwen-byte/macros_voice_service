# Mac 系统音频识别工具

基于 FunASR paraformer-zh 模型的语音识别工具，**将系统播放的声音实时转换为文字**。

## 🎯 功能特性

- ✅ **系统内录**：支持 B 站视频、会议录音、在线课程等所有系统播放的音频
- ✅ **图形界面**：双击启动，一键开始/停止，无需命令行操作
- ✅ **实时显示**：双标签页设计，模型加载时看日志，完成后看转录内容
- ✅ **自动检测**：静音过滤，避免垃圾输出
- ✅ **文件保存**：按时间戳自动保存转录结果

---

## 🚀 快速开始

### 方式 1: 图形界面（推荐）

```bash
cd asr_project
python3 gui_controller.py
```

**操作步骤：**
1. 点击 **"▶️ 开始识别"** 按钮
2. 等待模型加载（首次约 2-3 分钟）
3. 界面自动切换到 **实时转录** 页面
4. 播放 B 站视频或其他音频即可看到文字
5. 点击 **"⏹️ 停止识别"** 结束

### 方式 2: 命令行模式

```bash
# 启动服务
python3 asr.py start

# 查看状态
python3 asr.py status

# 停止服务
python3 asr.py stop
```

### 方式 3: 一键重启脚本

```bash
./restart.sh
```

## 🎧 音频配置（重要）

要识别**系统播放的声音**（而非麦克风），需配置 **BlackHole 虚拟声卡**：

1. 打开「音频 MIDI 设置」
2. 创建**多输出设备**：勾选 **BlackHole 2ch** + **内建输出**
3. 在系统声音设置中选择该多输出设备

**测试方法**：播放 B 站视频，检查 ASR 是否能识别出字幕内容。

## ⚙️ 技术细节

| 参数 | 值 | 说明 |
|------|-----|------|
| **模型** | paraformer-zh | 中文优化版本 |
| **采样率** | 16kHz | 单声道 |
| **滑动窗口** | 10 秒 | 保证句子完整性 |
| **音量阈值** | 0.04 | 低于此值跳过识别 |
| **Python 环境** | QwenPaw 虚拟环境 | 避免污染系统 Python |

## ❓ 常见问题

**Q: 为什么每个字之间有空格？**  
A: FunASR 1.1.9 版本的输出格式限制，二期可升级到带标点版本。

**Q: 系统没有声音时输出了乱码？**  
A: 已添加音量阈值过滤。如果仍有问题，可调整 `asr_core.py` 第 102 行的 `volume_threshold`：
- 调高（0.06-0.10）→ 更严格，只识别大声内容
- 调低（0.02-0.03）→ 更宽松，小声也能识别

**Q: 可以识别英文吗？**  
A: 当前版本专注于中文。如需多语言支持，需切换到 multilingual 模型。

**Q: 支持哪些场景？**  
A: **任何系统播放的音频**：
- ✅ B 站/YouTube 视频
- ✅ 在线会议（腾讯会议/钉钉/Zoom）
- ✅ 网络课程（Coursera/Udemy/网易云课堂）
- ✅ 本地音视频播放器
- ✅ 播客/App 内的音频

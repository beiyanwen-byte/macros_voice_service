# Mac 系统音频识别工具（命令行版）

基于 FunASR paraformer-zh 模型的命令行语音识别工具，将**系统播放的声音**实时转换为文字。

## 🚀 快速开始

### 1. 启动服务

```bash
cd asr_project
python3 asr.py start
```

等待模型加载（首次约 2-3 分钟），显示提示后开始识别：

```
🚀 正在启动 ASR 服务...
📁 识别结果将保存到：asr_project/transcripts/2026-06-28_1450.txt
--------------------------------------------------
==================================================
✅ FunASR 模型已加载完成
==================================================
🎤 可以录制系统声音了！
```

**注意**：前台模式下按 `Ctrl+C` 即可停止。

### 2. 其他命令

```bash
# 查看服务状态
python3 asr.py status

# 停止服务
python3 asr.py stop

# 列出可用音频设备
python3 asr.py devices
```

### 3. 一键重启脚本

项目内提供了便捷的重启脚本：

```bash
cd asr_project
./restart.sh
```

该脚本会自动停止旧进程并启动新服务。

## 🎧 音频配置（重要）

要识别**系统播放的声音**（而非麦克风），需配置 **BlackHole 虚拟声卡**：

1. 打开「音频 MIDI 设置」
2. 创建**多输出设备**：勾选 **BlackHole 2ch** + **内建输出**
3. 在系统声音设置中选择该多输出设备

## ⚙️ 技术细节

- **模型**: paraformer-zh（中文优化）
- **采样率**: 16000Hz, 单声道
- **滑动窗口**: 10 秒（每次识别累积 10 秒音频）
- **Python 版本**: 3.12+
- **依赖环境**: 使用 QwenPaw 虚拟环境

## ❓ 常见问题

**Q: 为什么每个字之间有空格？**  
A: FunASR 1.1.9 版本在未启用标点模型时的输出格式。二期可升级到支持标点的版本。

**Q: 首次启动很慢？**  
A: FunASR 模型首次加载需要 2-3 分钟，后续启动会很快。

**Q: 系统没有声音时输出了乱码？**  
A: 已添加音量阈值过滤（v1.1+）。检查终端日志中的"检测到音量"提示，如果频繁跳过识别，可以尝试调低阈值。

**Q: 如何修改识别间隔？**  
A: 编辑 `asr_core.py` 第 98 行的 `buffer_duration = 10.0` 参数。

**Q: 如何调整音量阈值？**  
A: 编辑 `asr_core.py` 第 102 行的 `volume_threshold = 0.04`：
- 调高（0.06-0.10）→ 更严格，只识别大声内容
- 调低（0.02-0.03）→ 更宽松，小声也能识别

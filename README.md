# Mac 系统音频识别工具（命令行版）

基于 FunASR paraformer-zh 模型的命令行语音识别工具，将**系统播放的声音**实时转换为文字。

## 快速开始

### 1. 启动服务（首次需等待模型加载约 2-3 分钟）

```bash
cd asr_project
python3 asr.py start
```

等待显示：
```
🚀 正在启动 ASR 服务...
📁 识别结果将保存到：asr_project/transcripts/2026-06-28_1450.txt
--------------------------------------------------
==================================================
✅ FunASR 模型已加载完成
==================================================
🎤 可以录制系统声音了！
--------------------------------------------------
请确保:
  1. 系统音频输出已路由到 BlackHole 虚拟声卡
  2. 播放音频即可实时识别为文字
--------------------------------------------------
```

**注意**：启动后会持续运行，直到你按 `Ctrl+C` 停止。实时识别的日志会显示在终端。

### 2. 停止服务

在服务运行时按 `Ctrl+C`，或开新窗口执行：
```bash
python3 asr.py stop
```

## 音频配置（重要）

要识别**系统播放的声音**（而非麦克风），需配置 **BlackHole 虚拟声卡**：

1. 打开「音频 MIDI 设置」
2. 创建**多输出设备**：勾选 **BlackHole 2ch** + **内建输出**
3. 在系统声音设置中选择该多输出设备

```bash
# 查看可用设备
python3 asr.py devices
```

## 其他命令

```bash
# 查看服务状态
python3 asr.py status

# 列出音频输入设备
python3 asr.py devices
```

## 输出文件格式

每次启动都会创建新的文件，格式如下：

```txt

# 输出文件：asr_project/transcripts/2026-06-28_1449.txt
# 音频设备：Built-in Microphone

[14:50:24] 力啊

[14:50:26] 这个书写
[14:50:27] 问题
[14:50:29] 少要么
```

## 技术细节

- **模型**: paraformer-zh（中文优化）
- **采样率**: 16000Hz, 单声道
- **识别间隔**: 500ms
- **依赖环境**: 使用 QwenPaw 虚拟环境（避免污染系统 Python）

## 常见问题

**Q: 为什么每个字之间有空格？**  
A: FunASR 1.1.9 版本在未启用标点模型时的输出格式。二期可升级到支持标点的版本。

**Q: 首次启动很慢？**  
A: FunASR 模型首次加载需要 2-3 分钟，后续启动会很快。

**Q: 没有识别到声音？**  
A: 请确认系统音频输出已路由到 BlackHole 虚拟声卡（见上方「音频配置」）。
